> **Propósito**: comparar cuatro librerías NuGet de escaneo de QR/códigos de barras en .NET MAUI (BSM/BSN/CS/ZN), cada una en dos variantes de UX (página embebida vs. diálogo con retorno de resultado).
> **Fuente primaria**: `Ejemplos_Devices/QR/`.

## 3. Motor de decodificación por plataforma

Es el eje real de la comparación: qué API nativa usa cada librería y su impacto en simulador iOS / Rosetta (`Docs/qr-nuget/02-alternativas-nuget.md`, `03-matriz-rid.md`).

| Prefijo | Android | iOS | Decodificación | Simulador iOS arm64 | Rosetta / `iossimulator-x64` |
|---|---|---|---|---|---|
| **BSM** | ML Kit (CameraX) | **ML Kit** | nativa Google | ⚠️ sin slice sim arm64 | **requerido** (workaround) |
| **BSN** | ML Kit (CameraX) | **Apple Vision** | nativa | ✅ funciona | **no** necesita |
| **CS** | ML Kit (CameraX) | **Apple VisionKit** | nativa | ⚠️ parcial | evita ML Kit iOS |
| **ZN** | Camera2 | AVFoundation | **managed ZXing** | ✅ (managed) | no aplica |

> **Hallazgo central del proyecto** (`Docs/qr-nuget/README.md`, `01-analisis-uso-actual.md:147-154`): BSM (`BarcodeScanner.Mobile.Maui`) usa **ML Kit también en iOS**, que no publica slice de simulador `arm64` → obliga al workaround `iossimulator-x64` + **Rosetta** (en retirada por Apple). Las familias BSN/CS/ZN existen como **candidatas de migración** para eliminar esa dependencia; BSN (Apple Vision) es la recomendación #1 del doc `04-recomendacion.md`.

### 4.3 Mapeo del resultado a texto

Convención común en página (muestra en `DisplayAlert`) y diálogo (rellena `QRContent`): el tipo se normaliza a `"Text"` cuando el enum es `Unknown`.

| Prefijo | Expresión de mapeo | Fuente |
|---|---|---|
| BSM (página) | `$"Type: {b.BarcodeType}, Value: {b.DisplayValue}"` | `BSM.LectorQR/Pages/MainPage.xaml.cs:53` |
| BSN / CS (página) | `$"{(b.BarcodeType==Unknown?"Text":...)}: {b.DisplayValue}"` | `BSN.LectorQR/Pages/MainPage.xaml.cs:46` |
| ZN (página) | `$"{r.BarcodeFormat}: {r.Text}"` | `ZN.LectorQR/Pages/MainPage.xaml.cs:42` |

## 6. Variante Página vs. Diálogo

Ambas variantes de cada familia comparten **exactamente** el mismo control, motor y API; solo cambia el patrón de UX y de retorno del resultado.

| Aspecto | `*.LectorQR` (Página) | `*.LectorQR_Dialog` (Diálogo) |
|---|---|---|
| **Punto de entrada** | `MainPage` **es** el escáner | `MainPage` con botón "Leer QR" → navega |
| **Página de escaneo** | `Pages/MainPage.xaml(.cs)` | `Pages/QRLectorPage.xaml(.cs)` (dedicada) |
| **Al detectar** | `DisplayAlert` inline y **reanuda** escaneo | mapea a `QRContent`, **cierra** y devuelve resultado |
| **Retorno de datos** | ninguno (se muestra en la misma página) | `TaskCompletionSource<List<QRContent>>` (`ResultadoTask.Task`) |
| **Navegación** | ninguna | `Navigation.PushAsync` / `PopAsync` (dentro de `Shell`) |
| **Modelo de datos** | — | `Models/QRContent.cs` (`{ Type, Value }`) |
| **Cancelación** | — | botón "Volver" → resultado vacío + `PopAsync` |
| **Orientación** | estática (`Grid` fijo) | **layout dinámico** portrait/landscape recalculado |
| **Iconografía** | botones de texto | fuente `MaterialIconsOutlined` (flash/back) |
| **Fuente extra** | — | `MaterialIconsOutlined-Regular.otf` (`BSN.LectorQR_Dialog/MauiProgram.cs:18`) |

Fuentes página: `BSM.LectorQR/Pages/MainPage.xaml.cs:56-61` (alert + reanuda). Fuentes diálogo: `BSM.LectorQR_Dialog/Pages/MainPage.xaml.cs:12-39`, `QRLectorPage.xaml.cs`.

### 6.1 Flujo de la variante Diálogo

```mermaid
sequenceDiagram
    participant U as Usuario
    participant M as MainPage
    participant Q as QRLectorPage
    participant L as Librería (CameraView)
    U->>M: Tap "Leer QR"
    M->>Q: Navigation.PushAsync(new QRLectorPage())
    M->>M: await destinoPage.ResultadoTask.Task
    Q->>Q: OnAppearing → pide permiso cámara
    L-->>Q: evento detección (BarcodeResults / Result)
    Q->>Q: map → List<QRContent>; CompletarResultado()
    Q->>M: Navigation.PopAsync()  (TCS resuelto)
    M->>U: LbQR.Text = qr.Value  (o "Cancelado")
```

### 6.2 Patrón interno del Diálogo (idéntico en las 4 familias)

- **Retorno seguro**: `TaskCompletionSource<List<QRContent>>` + guarda `Interlocked.Exchange(ref _completed, 1)` para completar **una sola vez** (detección o "Volver"). Fuente `BSM.LectorQR_Dialog/Pages/QRLectorPage.xaml.cs:11-12,136-142`.
- **Layout adaptativo**: `UpdateLayoutOrientation()` reconstruye `RowDefinitions`/`ColumnDefinitions` del `Grid` (`DynamicLayout`) en portrait vs. landscape, suscrito a `DeviceDisplay.MainDisplayInfoChanged` (alta en `OnAppearing`, baja en `OnDisappearing`). Fuente `QRLectorPage.xaml.cs:150-213`.
- **Flash con icono dinámico**: `FlashIcon` (`flash_on`/`flash_off`) vía `OnPropertyChanged` + `FontImageSource` Material. Fuente `QRLectorPage.xaml:18-20`, `.xaml.cs:144-148`.
- **Modelo**: `QRContent { string Type; string Value; }` — POJO sin dependencia de la librería, lo que hace la migración localizada. Fuente `BSM.LectorQR_Dialog/Models/QRContent.cs`.

> El diálogo **no** usa `CommunityToolkit.Maui.Popup`: es una `ContentPage` completa apilada por `Shell`/`Navigation` que simula un modal y devuelve datos por `Task`.

## 8. CI/CD y errores de compilación iOS

Documentado en `Ejemplos_Devices/QR/Ejemplo_Docs_QR/cicd.md` y `Readme.md` (foco en el pipeline GitHub Actions con Xcode 26 / .NET 10.0.102).

| Síntoma | Causa | Mitigación documentada |
|---|---|---|
| `ld: building for 'iOS-simulator', but linking object built for 'iOS'` en `MLKitCommon` | ML Kit publica frameworks solo para device; el simulador arm64 no linkea | `MtouchExtraArgs` con `--setenv:DYLD_LIBRARY_PATH=/usr/lib/system/introspection` + `-gcc_flags "-Wl,-ignore_auto_link -Wl,-read_only_relocs,suppress"`, `MtouchLink=SdkOnly`, `NoWarn=MT5209;MT5212;MT5213` (`cicd.md:248-254`) |
| `NETSDK1084: no application host for RuntimeIdentifier 'android-arm'...` durante restore iOS | ruido de multi-TFM al restaurar android RIDs en build iOS | informativo; no bloquea (`cicd.md:103-106`) |
| Conflictos de librerías nativas Google (MLKit + Firebase) | componentes duplicados | `--ignore-native-library-conflict` + `NoWarn=MT5209;MT5212` (`Readme.md:66-74`) |

**Recomendación de pipeline** (`Readme.md:90-103`): para CI que solo verifica compilación o genera IPA de testing, **no** usar simulador; compilar contra hardware real `-p:RuntimeIdentifier=ios-arm64 -p:BuildIpa=true -p:EnableCodeSigning=false`.

> El `cicd.md` referencia el proyecto histórico `Ejemplo_LectorQR_Dialog` (`PACKAGE_NAME=com.ejemplos.devices.qr.dialog`), predecesor de la división actual en BSM/BSN/CS/ZN.
